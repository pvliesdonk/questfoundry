# Generated Code: NEVER EDIT DIRECTLY

**This rule is enforced by hook. Violations will be blocked.**

## The Rule

Files in `src/questfoundry/generated/` are AUTO-GENERATED by the compiler.

- **NEVER** edit files in `generated/` directly
- **ALWAYS** edit source files in `domain/` and run `qf compile`

## Source-to-Generated Mapping

| Source | Generated |
|--------|-----------|
| `domain/roles/*.md` | `generated/roles/*.py` |
| `domain/ontology/artifacts.md` | `generated/models/artifacts.py` |
| `domain/ontology/enums.md` | `generated/models/enums.py` |

## When You See a Bug in Generated Code

1. **Find the source** - Check corresponding `domain/` file
2. **Fix the source** - Edit the `.md` file OR fix the compiler
3. **Regenerate** - Run `qf compile`
4. **Verify** - Check `git diff src/questfoundry/generated/`

## Why This Matters

- Edits to `generated/` will be overwritten on next compile
- Previous sessions have repeatedly made this mistake
- The hook exists because instructions alone don't prevent this
