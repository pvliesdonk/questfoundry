# World Genesis

> Build the world before the story. This playbook initializes a new story project.

This playbook defines the procedure for setting up a new story world, establishing foundational canon before plot development begins.

## When to Use

Invoke this playbook when:

- Starting a new story project from scratch
- Setting requires substantial worldbuilding (epic fantasy, deep sci-fi)
- Canon foundation needed before topology design
- Team needs shared understanding of world constraints

**Do NOT use for:**

- Adding content to existing world (use standard loops)
- Small-scale stories with minimal worldbuilding
- Projects starting from existing source material

## Prerequisites

Before starting World Genesis:

- [ ] Project concept defined (genre, setting, scope)
- [ ] Worldbuilding scope agreed (which themes to cover)
- [ ] Style preferences outlined (voice, register, motifs)
- [ ] Team roles identified (which are active)

## Initialization Steps

### Step 1: Scope Definition (Showrunner)

Define worldbuilding parameters:

1. **Themes to cover**: geography, magic/metaphysics, history, factions, culture
2. **Budget level**: minimal (1-2 hours), standard (4-8 hours), epic (20+ hours)
3. **Depth per theme**: surface-level vs. deep detail
4. **Heuristic**: Stop when Plotwright's likely questions are answerable

### Step 2: Create Initial Structure

Set up cold_store foundation:

1. World metadata (name, genre, tone)
2. Empty containers for each theme
3. Style guide placeholder
4. Timeline foundation (T0 = story present)

### Step 3: Worldbuilding Execution

**For each scoped theme:**

#### Geography

- Regions, landmarks, travel routes
- Climate and environmental constraints
- Invariants: "The Ironwood Forest is impassable in winter"

#### Magic/Metaphysics (if applicable)

- Rules, limits, sources
- Who can use it, how it's acquired
- Invariants: "Magic cannot resurrect the dead"

#### History

- Timeline depth (how far back)
- Major events, dynasties, cataclysms
- Invariants: "The Blood Compact binds all signatories"

#### Factions

- Major powers, structures, goals
- Rivalries and alliances
- Invariants: "House Vaelen controls northern trade"

#### Culture/Technology (optional)

- Technology baseline
- Customs, language notes
- Social structures

### Step 4: Style Anchors (Creative Director)

Establish voice and tone:

- Narrative perspective (person, tense)
- Formality level
- Recurring motifs
- Dialogue patterns

### Step 5: Validation (Gatekeeper)

Before completing genesis:

- **Integrity**: All references resolve within genesis canon
- **Style**: Voice consistent across genesis content
- **Presentation**: Player-facing content is spoiler-safe

Note: Reachability/Nonlinearity/Gateways don't apply yet (no plot).

### Step 6: Create Constraint Manifest

Document for Plotwright:

**Invariants (cannot change):**

- List hard constraints from each theme
- These are non-negotiable in plot development

**Affordances (available for use):**

- Locations that can be story hubs
- Factions that can drive conflict
- Magic/tech that enables plot devices

**Can/Cannot lists:**

- Explicit guidance on what Plotwright may and may not do

## Worldbuilding Budget Guidance

### Minimal Genesis (1-2 hours)

- 1-2 theme areas covered
- 5-10 canon entries
- Basic style notes
- 3-5 timeline anchors

### Standard Genesis (4-8 hours)

- 3-5 theme areas covered
- 20-30 canon entries
- Full style addendum
- 5-10 timeline anchors

### Epic Genesis (20+ hours)

- All theme areas deeply covered
- 100+ canon entries
- Multiple style variations (per region/faction)
- 50+ timeline anchors

## Deliverables

At end of World Genesis:

- [ ] Canon entries for each scoped theme
- [ ] Style addendum (voice, motifs, register)
- [ ] Timeline foundation
- [ ] Constraint manifest for Plotwright
- [ ] Gatekeeper validation report

## Handoffs

World Genesis hands off to:

- **Story Spark**: Constraint manifest, timeline, style guide
- **Lorekeeper**: Canon entries become the initial cold_store
- **Plotwright**: Affordances and constraints for topology design

## Success Criteria

Genesis is complete when:

- [ ] All scoped themes have canon coverage
- [ ] No internal contradictions
- [ ] Style is defined and consistent
- [ ] Constraint manifest is actionable
- [ ] Gatekeeper approves
- [ ] Plotwright can begin topology design

## Escape Hatches

**Canon sprawl** (too much worldbuilding)

- Set budget and stick to it
- Defer non-critical themes to "extend during later loops"
- Heuristic: Stop when Plotwright's questions are answerable

**Internal contradictions**

- Lorekeeper identifies and resolves
- Gatekeeper blocks until resolved
- May need to choose between conflicting ideas

**Over-building**

- Not all canon will be used—that's okay
- Better to have constraints than to invent mid-story
- Unused canon can inform future expansions

## Summary

World Genesis builds the foundation that all future content rests on. Be thorough enough to constrain and enable Plotwright, but not so thorough that you never start the actual story. The world serves the story, not the reverse.
